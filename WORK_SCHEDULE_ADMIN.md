# Управление графиком работы

## Описание

Система работает с настраиваемым графиком рабочего времени, что позволяет:
- Корректно считать сроки проектов, пропуская выходные
- Расчитывать длительность задач в рабочих днях (не календарных)
- Автоматически применять множитель очков геймификации в зависимости от дня недели
- Правильно планировать ревью документов и проверки

## По умолчанию

Система работает с такими настройками:
- **Рабочая неделя**: Понедельник - Пятница (5 дней)
- **Рабочие часы**: 8:30 - 17:30
- **Перерыв на обед**: 1 час (бесплатный)
- **Рабочих часов в день**: 8 часов (17:30 - 8:30 - 1 час обеда)
- **Рабочих часов в неделю**: 40 часов

## API для администратора

### 1. Получить все графики работы
```
GET /api/admin/work-schedules/
```

Возвращает список всех доступных графиков с информацией о:
- Названии
- Рабочих днях недели
- Времени начала/конца
- Длительности обеда
- Ежедневное и еженедельное количество рабочих часов

### 2. Получить стандартный график (public)
```
GET /api/admin/work-schedules/default
```

Возвращает текущий действующий календарь (используется для расчетов сроков).

### 3. Создать новый график
```
POST /api/admin/work-schedules/
{
  "name": "Гибкий график 4/10",
  "work_days": [0, 1, 2, 3],  // Пн-Чт (4 дня)
  "start_time_hours": 9.0,     // 9:00
  "end_time_hours": 19.0,      // 19:00
  "lunch_duration_hours": 1.0,
  "is_default": true
}
```

**Параметры:**
- `name`: Название графика (для отбора в UI)
- `work_days`: Массив дней недели (0=Пн, 1=Вт, ..., 6=Вс)
- `start_time_hours`: Время начала (дробные часы, напр. 8.5 = 8:30)
- `end_time_hours`: Время окончания
- `lunch_duration_hours`: Длительность обеда в часах
- `is_default`: Если true, этот график становится главным для расчетов

Когда вы создаете новый график как `is_default=true`, остальные автоматически перестают быть стандартными.

### 4. Обновить график
```
PUT /api/admin/work-schedules/{schedule_id}
{
  "name": "Обновленный график",
  "start_time_hours": 8.0,
  // ... прочие поля опционально
}
```

### 5. Расчитать часы в дни (для планирования)
```
POST /api/admin/work-schedules/calculate/hours-to-days?work_hours=40
```

Ответ:
```json
{
  "work_hours": 40,
  "work_days": 5,
  "work_hours_per_day": 8
}
```

### 6. Расчитать дни в часы
```
POST /api/admin/work-schedules/calculate/days-to-hours?work_days=10
```

Ответ:
```json
{
  "work_days": 10,
  "work_hours": 80,
  "work_hours_per_day": 8
}
```

### 7. Определить дату окончания задачи
```
POST /api/admin/work-schedules/calculate/end-date?start_date=2026-04-06&work_days=5
```

Ответ:
```json
{
  "start_date": "2026-04-06",  // Понедельник
  "work_days_duration": 5,
  "end_date": "2026-04-10",    // Пятница (пропущены выходные)
  "actual_work_days": 5
}
```

## Примеры настроек

### Стандартный график (по умолчанию)
```json
{
  "name": "Стандартный график (Пн-Пт 8:30-17:30)",
  "work_days": [0, 1, 2, 3, 4],
  "start_time_hours": 8.5,
  "end_time_hours": 17.5,
  "lunch_duration_hours": 1.0,
  "is_default": true
}
```
Результат: 40 часов в неделю (8 часов × 5 дней)

### Гибкий график 4/10 (4 дня по 10 часов)
```json
{
  "name": "Гибкий график 4/10",
  "work_days": [0, 1, 2, 3],
  "start_time_hours": 8.0,
  "end_time_hours": 18.0,
  "lunch_duration_hours": 1.0,
  "is_default": false
}
```
Результат: 36 часов в неделю (9 часов × 4 дня)

### Суббота - полный день
```json
{
  "name": "Пн-Сб с выходным в среду",
  "work_days": [0, 1, 3, 4, 5],
  "start_time_hours": 8.5,
  "end_time_hours": 17.5,
  "lunch_duration_hours": 1.0,
  "is_default": false
}
```
Результат: 40 часов в неделю

### Летнее время (сокращенный день в пятницу)
```json
{
  "name": "Летний график (Пт с 8:30 до 13:00)",
  "work_days": [0, 1, 2, 3, 4],
  "start_time_hours": 8.5,
  "end_time_hours": 13.0,
  "lunch_duration_hours": 0.0,
  "is_default": false
}
```
Результат: 4.5 часа в день × 5 дней = 22.5 часов

## Интеграция в системе

### Геймификация
- При расчете комбо-множителя используется информация о дне недели
- Рабочие дни могут получить дополнительный множитель

### Планирование задач
- При создании задачи с указанием часов система автоматически конвертирует в рабочие дни
- При расчете CPM используются только рабочие дни (выходные пропускаются)
- Дата окончания задачи автоматически сдвигается на следующий рабочий день, если приходится на выходной

### Уведомления
- Система может отправлять уведомления только в рабочие дни
- Крайние сроки расчитаются с учетом выходных

## Технические детали

### API Endpoints
```
GET    /api/admin/work-schedules/               — Список графиков
GET    /api/admin/work-schedules/default        — Стандартный график
POST   /api/admin/work-schedules/               — Создать график
PUT    /api/admin/work-schedules/{id}           — Обновить график
POST   /api/admin/work-schedules/calculate/hours-to-days      — Часы → Дни
POST   /api/admin/work-schedules/calculate/days-to-hours      — Дни → Часы
POST   /api/admin/work-schedules/calculate/end-date           — Дата окончания
```

### Модели

**WorkSchedule:**
- `id`: Integer
- `name`: Text
- `work_days`: JSON Array [0-6] (0=Monday, 6=Sunday)
- `start_time_hours`: Numeric(4,2)
- `end_time_hours`: Numeric(4,2)
- `lunch_duration_hours`: Numeric(4,2)
- `is_default`: Boolean

### Сервисы
- `WorkScheduleRepository`: CRUD операции
- `TimeCalculationService`: Конверсия между часами и днями, расчет дат
